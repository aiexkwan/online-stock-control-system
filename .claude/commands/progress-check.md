# Progress Check Command

## 用法
`/check` 或 `/check [功能/任務名稱]`

## 執行流程
1. **啟動工具**
   - Ultrathink - 深層思考分析
   - Sequential-thinking - 系統性檢查
   - Task - 同步平行執行驗證

2. **閱讀基礎文檔**
   - 讀取 `docs/general_rules.md` 了解規範
   - 檢查相關計劃文檔

3. **角色建議**
   - 閱讀 docs\role_play 內的角色文檔
   - 主要角色: 📊 Analyzer（進度分析）
   - 協作角色: ⚙️ Backend + ✅ QA
   - 替代角色: 根據關注點選擇

4. **進度檢查步驟**
   - 查看 `docs/Today_Todo/` 的當天進度
   - 對比 `docs/planning/` 的原始計劃
   - 檢查 `docs/progress-check/` 的歷史記錄

5. **實際驗證**
   - 使用 Playwright 進行功能測試
   - 使用 Vitest 運行單元測試
   - 使用 Puppeteer MCP 進行視覺驗證

6. **合規檢查**
   - 確認是否按照計劃執行
   - 檢查是否遵守編碼原則
   - 驗證 TypeScript 和 ESLint 無錯誤

## 檢查項目清單
```yaml
功能完成度:
  - 計劃功能是否全部實現
  - 是否有遺漏的邊緣案例
  - 錯誤處理是否完善

代碼品質:
  - TypeScript 類型檢查通過
  - ESLint 規則符合
  - 無冗餘代碼

測試覆蓋:
  - 單元測試是否完整
  - E2E 測試是否通過
  - 關鍵路徑是否測試

文檔更新:
  - 相關文檔是否同步更新
  - API 文檔是否準確
  - 用戶手冊是否完整
```

## 相關命令
- `/plan` - 查看原始計劃
- `/start` - 繼續執行任務
- `/audit` - 進行代碼審核
