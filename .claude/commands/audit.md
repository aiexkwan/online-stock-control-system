# Audit Command

## 用法
`/audit [檔案/模組/功能]`

## 審核項目
1. ✅ 重複或不合理的讀寫操作
2. ✅ 循環引用和相互依賴
3. ✅ Edge case 和錯誤處理機制
4. ✅ 冗碼和過時註釋
5. ✅ 編碼原則遵守情況
6. ✅ 用戶操作流程順暢度

## 角色建議
- 閱讀 docs\role_play 內的角色文檔
- 主要角色: 🧹 Refactor（代碼品質）
- 協作角色: 🔒 Security + ⚡ Optimizer
- 專項審核: 針對前端、後端、流程等

## 執行流程
1. 閱讀相關文檔和代碼
2. 建立審核 todolist
3. 使用 Playwright 測試驗證
4. 報告輸出路徑: docs/audit/[功能]-v[X.X.X].md

## 審核標準
參考 docs/general_rules.md 的編碼原則
